Linux容器实现原理

Docker是一种容器，容器本身的价值不高，有价值的是容器编排、融合和均衡技术
构建“容器”隔离环境的技术：Namespace、Cgroup、rootfs

Namespace：“隔离“作用，使进程只能看到隔离内的世界；其是修改进程视图的主要方法
Cgroup：”限制“作用，给世界围上一圈看不见的墙，约束进程的主要手段
rootfs：一组联合挂载在 /var/lib/docker/aufs/mnt 上的 rootfs，这一部分我们称为“容器镜像”（Container Image），是容器的静态视图；一个由 Namespace+Cgroups 构成的隔离环境，这一部分我们称为“容器运行时”（Container Runtime），是容器的动态视图

Docker引擎搭建

Linux系统 -uname -a 版本在3.6以上
1.getenforce查看SElinux策略(vi /etc/selinux/config 切换关闭)

2.安装Base源 curl -o /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo

3.安装epel源 yum install epel-release -y

4.列出docker可安装的软件包 yum list docker –show-duplicates

5.使用yum-config-manager –add-repo前置条件安装yum install -y yum-utils

6.阿里云docker-ce yum仓库

yum-config-manager –add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

7.列举docker-ce文件 yum list docker-ce –show-duplicates

8.安装最新的docker-ce源 yum install docker-ce

9.将docker设定为开机自启动 systemctl enable docker

10.启动docker systemctl start docker

11.在etc下创建docker目录 mkdir /etc/docker

12.配置docker引擎 vi /etc/docker/daemon.json

{
“graph”:”/data/docker”,
“storage-driver”:”overlay2”，
“insecure-registries”:[“registry.access.redhat.com”,”quay.io”],
“registry-mirrors”:[“https://alzgoonw.mirror.aliyuncs.com"],
“bip”:”172.20.251.1/24”,
“exec-opts”:[“native.cgroupdriver=systemd”],
“live-restore”:true
}

*注意，以上docker引擎配置文件末尾需加英文状态中的逗号

–graph 目录
–storage-driver 存储驱动
–insecure-registries docker客户端如果配置中添加了insecure-registary配置，就不需要在docker 客户端配置上对应证书
–registry-mirrors 镜像
–bip docker 网络地址
–exec-opts
–live-restore [true] 容器引擎宕机不影响利用docker引擎启动的容器

重新加载daemon.json配置 systemctl daemon-reload

Docker 启动

docker run hello-world
重点：

To generate this message, Docker took the following steps:

  1. The Docker client contacted the Docker daemon.
  2. The Docker daemon pulled the “hello-world” image from the Docker Hub.(amd64)
  3. The Docker daemon created a new container from that image which runs the executable that produces the output you are currently reading.
  4. The Docker daemon streamed that output to the Docker client, which sent it to your terminal.

Docker容器基础操作

1.查看本地docker容器进程 docker ps -a

2.先查询镜像 docker images
交互式启动需求镜像容器 docker run -ti boydocker8899/alpine:v3.10.3 /bin/sh

3.非交互式启动需求镜像容器 docker run –rm boydocker8899/alpine:v3.10.3 /bin/echo hello

4.非交互式启动后台镜像容器 docker run -d –name myalpine boydocker8899/alpine:v3.10.3 /bin/sleep 300 (300为睡眠时间)

5.查看宿主进程 ps aux|grep sleep|grep -v grep

停止容器 docker stop ID/NAME

8.过滤出宿主机上所有退出的容器并删除脚本

for i in docker ps -a|grep -i exit|awk '{print $1}';do docker rm -f $i;done

9.容器可写层中写入文件，容器关闭后，文件消失。若想保存文件，需将文件写进镜像中

10.写镜像 docker commit -p myalpine boydocker8899/alpine:v3.10.3_with_1.txt

11.导出镜像 docker save dd3350829a49 > alpine:v3.10.3_with_1.txt.tar

​ 导入镜像 docker load < alpine:v3.10.3_with_1.txt.tar

12.查看容器日志

​ docker run hello-world 2>&1 >>/dev/null

Dockerfile

1.USER/WORKDIR指令

root@localhost ~]# cd /data/
[root@localhost data]# mkdir dockerfile
[root@localhost data]# cd ~
[root@localhost ~]# cd /data/dockerfile/
[root@localhost dockerfile]# vi Dockerfile

FROM stanleyws/nginx:v1.12.2
USER nginx
WORKDIR /usr/share/nginx/html

构建镜像 docker build . -t docker.io/boydocker8899/nginx:v1.12.2_with_user_workdir /bin/bash

netstat工具包安装 yum install net-tools

Docker 容器启动时，默认会把容器内部第一个进程，也就是pid=1的程序，作为docker容器是否正在运行的依据，如果 docker 容器pid=1的进程挂了，那么docker容器便会直接退出。

Docker未执行自定义的CMD之前，nginx的pid是1，执行到CMD之后，nginx就在后台运行，bash或sh脚本的pid变成了1。

所以一旦执行完自定义CMD，nginx容器也就退出了。

3.RUN/ENV指令

bind软件包版本查看 yum list bind –show-duplicates

FROM centos:7
ENV VER 9.11.4
RUN yum install bind-$VER -y

docker build . -t boydocker8899/bind:9.11.4_with_env_run
